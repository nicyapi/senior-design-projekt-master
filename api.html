/api/events get post

get

Get all events of current user

post

Add an event to the user's list of events

get /api/events

Get all events of current user

  • Request
  • Response

Query Parameters

  • day: required(datetime | date)

    Filters the list of events and returns all the events from 30 days before, and after the provided date. If not provided, defaults to the current day.

  • activityId: required(integer)

    "Filters the list of events and returns all the events belonging to activity of id: activityId"

HTTP status code 200

Successfully retrieved event list

Body

Media type: application/json

Type: array of EventCreate

Items: Event

  • title: required(string)
  • summary: (string)
  • start: required(datetime)
  • end: required(datetime)
  • location: (nil)
  • activity: (union of integer or nil)

    The id of the category for which the activity belongs

  • id: required(integer)
  • user: required(integer)

Example:

[
  {
    "title": "Work on Homework",
    "summary": "Boo homework",
    "start": "2020-02-11T09:30:00.000000Z",
    "end": "2020-02-11T07:00:00.000000Z",
    "location": "????????????????",
    "id": 1,
    "user": 1,
    "activity": 1
  },
  {
    "title": "Class",
    "summary": "",
    "start": "2020-02-11T09:30:00.000000Z",
    "end": "2020-02-11T07:00:00.000000Z",
    "location": "????????????????",
    "id": 2,
    "user": 1,
    "activity": null
  }
]

HTTP status code 401

The user is not signed in
